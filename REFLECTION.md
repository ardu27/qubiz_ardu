# Reflection

Here are a few honest thoughts looking back on building this project.

### What turned out to be harder than you expected?

Honestly, the hardest part was just finding the time to actually sit down and start coding. When I got the email, I was right in the middle of my university exams, so everything got pushed back a bit. Juggling studying for finals and trying to focus on building a whole full-stack app was definitely a lot harder than I anticipated.

### Which decision would you make differently if you started over?

If I had started right away when the email came, I would have definitely tried a completely new architecture, just to learn something fresh. But since I started late, I had to be realistic. I just went with .NET and React because that's what I knew how to piece together quickly. Splitting the backend into microservices was maybe a bit "extra" for a project this size, but I figured it would be fun to try, even if a simple monolith would have been way less of a headache to set up.

### What did you learn about yourself as a developer during this project?

I learned a really hard lesson about time management and scope creep. When you're brainstorming a project and bouncing ideas off classmates, it's so easy to say, "Oh, it would be awesome if the app also did X and Y!" I realized I have a tendency to want to build every cool feature that pops into my head. Because I was strapped for time, I learned how to be ruthless with prioritization—ignoring the "nice-to-haves" and focusing strictly on the core features a new hire actually needs on day one.
